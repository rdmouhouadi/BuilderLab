# Contributing to BuilderLab

We welcome contributions from students, developers, and educators to improve the student collaboration platform!  
Please read the guidelines below to make contributing smooth and effective.

## How to Contribute
1. **Fork the repository** and create a new branch for your feature or fix.
2. **Implement your changes** according to our coding standards.
3. **Test your changes** to make sure everything works as expected.
4. **Submit a pull request (PR)** to the `main` branch with a clear description of your changes.

## Contributor License Agreement (CLA)
All contributors must agree to the [CLA](CLA.md). By submitting a PR, you confirm that you agree to the CLA.

## Good First Issues
We label beginner-friendly issues as `good first issue`. Look for these to get started quickly.

## Code of Conduct
We want the community to be inclusive and respectful. Please read our [Code of Conduct](CODE_OF_CONDUCT.md) and follow it at all times.

## Recognition
Top contributors may be highlighted in the project’s README or newsletters. Your work will be visible to students and future recruiters!
